Events
Event handlers in React
In React, you pass functions as props (like onClick) straight onto elements — no addEventListener needed. Your handler gets an event object; use event.target.value for inputs, for example.
// vanilla JS — attach in script
btn.addEventListener('click', handleClick);

// React — handler on the JSX element
<button onClick={handleClick}>
  Click me
</button>

// inline arrow — fine for one-liners
<button onClick={() => setCount(c => c + 1)}>
  +1
</button>
Common event props
  • onClick — button, div, any element
  • onChange — input, select, textarea
  • onSubmit — form
  • onFocus / onBlur — focus events
  • onKeyDown — keyboard
The event object
Handlers receive an event argument. Use event.target.value for inputs, event.preventDefault() to stop default browser behaviour.
Forms
Controlled inputs & onSubmit
const [name, setName] = useState('');

function handleSubmit(e) {
  e.preventDefault();
  // name is ready to use
}

return (
  <form onSubmit={handleSubmit}>
    <input
      value={name}
      onChange={e => setName(e.target.value)}
    />
    <button>Submit</button>
  </form>
);
Controlled inputs
The input's value is driven by state. Every keystroke fires onChange, which updates state, which re-renders. React owns the value.
Validation layers
HTML: required, minLength, type="email" — (easy, but limited)

JavaScript: validate inside handleSubmit for custom rules and error messages.

Server-side: always the final authority.
Hooks
Rules of Hooks
You already know one hook — useState. All hooks share the same two rules.
1 — Only call hooks at the top level
Never inside if, loops, or nested functions. React relies on the order of hook calls being the same every render.
2 — Only call hooks from React functions
Inside a component or a custom hook — never from plain JS functions.
// ✗ wrong — hook inside a condition
if (show) {
  const [x, setX] = useState(0);
}

// ✓ correct — always at the top level
const [x, setX] = useState(0);

if (show) {
  // use x here
}
Hooks
useEffect — run effect code after render
Use useEffect when you need to run side-effect code after render: timers, event listeners, network calls, browser APIs.
useEffect(() => {
  // runs after render
  const id = setInterval(() => {
    setTime(new Date());
  }, 1000);

  return () => clearInterval(id);
  // ↑ cleanup runs on unmount
}, []);
// ↑ [] = run once, on mount
Dependency array
No array: runs after every render.
[]: runs once on mount.
[a, b]: runs when a or b changes.
Cleanup function
Return a function to undo the effect — clear timers, remove event listeners, abort fetches. React calls it before the next run and when the component unmounts.
Loading Data
Fetching from an API
const [data, setData] = useState(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState(null);

useEffect(() => {
  fetch('https://api.example.com/data')
    .then((r) => r.json())
    .then((d) => { setData(d); setLoading(false); })
    .catch((e) => { setError(e); setLoading(false); });
}, []);

if (loading) return <p>Loading…</p>;
if (error) return <p>Error!</p>;
return <pre>{JSON.stringify(data)}</pre>;
Three states to handle
loading — show a spinner while waiting.

error — something went wrong; tell the user.

data — the happy path; render the result.
Why inside useEffect?
Fetching is a side effect — it's outside React's render. Putting it in useEffect with [] means it runs once after the component mounts.

Key takeaways
● Attach handlers as props — onClick, onChange, onSubmit — no addEventListener needed. In the form's onSubmit handler, call e.preventDefault() when you want to keep the user on the page and handle the submit in JavaScript (avoids a full page reload).
● Controlled inputs keep form data in state, giving you full control for validation and submission.
● Call hooks at the top level, never inside conditions or loops. React tracks them by order.
● useEffect runs side-effect code after render. Use the dependency array to control when it runs. Return a cleanup function when you set up timers or listeners.
● Fetch data inside useEffect with []. Track loading, error, and data as separate state variables.
