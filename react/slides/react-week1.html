Part 01 · What is a frontend framework?
Frontend libraries & frameworks
Tools that help you build UIs faster, with structure, reusability, and less repetition.
Without a framework
Raw HTML, CSS and JS. You manage everything yourself. Fine for small sites.
With a framework
You get conventions, patterns, and tools. Scales to big apps and big teams.

Part 02 · Why React?
You already know how to build things
HTML gives you structure: headings, divs, buttons
CSS gives you style: classes, selectors, layouts
JavaScript adds behavior: event listeners, DOM updates
So… what's the problem?
Part 02 · Why React?
Where it starts to break down
A lot of copy-pasting
If you need the same HTML in multiple places, you copy the code again and again.
Change it everywhere
If one thing changes, you have to update every copy in the codebase.
Manual DOM updates
You write lots of JavaScript just to keep the UI in sync with data.
Harder to maintain
As the codebase or team grows, changes become difficult to manage.
Part 02 · Why React?
What React gives you
Components
Build a piece of UI once. Reuse it anywhere, with no copy-pasting HTML ever again.
Automatic updates
When your data changes, the UI updates. No manual DOM work.
Structure for teams
Everyone works on separate components.
Strengths
+ Great for teams & big apps
+ Huge ecosystem & community
+ Design systems: write once
Trade-offs
– Overhead for tiny projects
– New concepts to learn
– Not a full framework (no router)
Part 03 · JSX
JSX: A new way to write your UI
Until now, you've kept your files separate.
index.html style.css script.js
In React, you write components in .jsx files, which let you write HTML-like syntax directly inside JavaScript.
Card.jsx App.jsx Header.jsx
JSX contains:
🏠 Structure: HTML-like syntax
💅🏼 Style: imported CSS or inline CSS-like syntax
⚙️ Behavior: JavaScript logic (variables, functions, etc.)
Part 03 · JSX
JSX fundamentals
function Greeting() {
  const name = 'HackYourFuture';
  const isSunday =
    new Date().getDay() === 0;

  return (
    <div>
      <h1>Hello {name}!</h1>
      {isSunday
        ? <p>Happy Sunday!</p>
        : null}
    </div>
  );
}
{ expressions }
Any valid JS: variables, ternaries, function calls
null renders nothing
Return null to hide an element conditionally
Must return one element
Everything must be wrapped in a single parent node
Components start with a capital letter
React uses capitalized names to indicate a component, lowercase for HTML elements
Part 03 · JSX
Browsers don't understand JSX
Browsers cannot run JSX directly. We need tools to turn it into regular JavaScript before the browser can run it.
We will use Vite for this
Vite starts your project and turns JSX into browser-ready JavaScript. While you work, it updates the app very quickly when you save.
Vite runs as a local development tool through Node.js.
Other options exist
Other tools exist, like Create React App, Webpack, and Parcel. Vite is the modern option, and it is what we will use in this session.

Part 03 · JSX rules
JSX uses camelCase attributes 🐪
In HTML
<button class="btn">

<label for="email">

<button onclick="fn()">
In React
<button className="btn">

<label htmlFor="email">

<button onClick={fn}>
HTML attributes that clash with JavaScript reserved words get renamed.
Part 03 · JSX rules
Everything needs a single root
Won't work
// two siblings — no parent
return (
  <h1>Title</h1>
  <p>Body</p>
);
Use a Fragment
// wraps without adding a DOM node
return (
  <>
    <h1>Title</h1>
    <p>Body</p>
  </>
);
<></> is a Fragment. It renders its children without adding an extra element to the DOM.
Part 04 · Components
What is a component?
A component is a reusable piece of UI.
Think of it like a custom HTML element you define once and use anywhere.
In React, a component is a function that returns JSX.
function Card() {
  return (
    <article>
      <h2>Title</h2>
      <p>Content</p>
    </article>
  );
}

// use it like an HTML tag
<Card />
Part 04 · Components
Components = UI building blocks 🧱
function Button() {
  return <button>Save</button>;
}

function Form() {
  return (
    <section>
      <input />
      <Button />
    </section>
  );
}

// small parts can build bigger screens

Reusable
Build a component once, then use it in many places across your app.
Combine them
A bigger UI can be made from smaller components working together.
They can be small or big
Components can be buttons, inputs, cards, forms, or entire pages.
